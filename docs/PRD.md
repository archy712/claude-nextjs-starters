# Notion 견적서 뷰어 MVP PRD

## 🎯 핵심 정보

**목적**: 사업자가 노션(Notion)에 입력한 견적서를, 클라이언트가 별도 가입 없이 고유 링크로 웹에서 확인하고 PDF로 다운로드할 수 있게 한다.
**사용자**: 노션으로 견적서를 작성·관리하는 1인/소규모 사업자(관리자)와, 그 사업자로부터 견적서 링크를 전달받는 고객(클라이언트).

## 🚶 사용자 여정

```
1. 첫 접속 ("/")
   ↓ 인증 상태 확인 (자동 리디렉션)

   [미인증] → 로그인 페이지
   [인증됨] → 관리자 대시보드
   ↓

2. 로그인 페이지
   ↓ 계정 없음 → 회원가입 페이지 → 가입 완료 → 로그인 페이지로 자동 이동
   ↓ 로그인 성공

3. 관리자 대시보드
   ↓ "노션 동기화" 클릭 → 노션 DB에서 견적서 목록 가져오기
   ↓ 견적서 항목의 "링크 복사" 클릭 → 고유 링크 클립보드 복사
   ↓ (앱 범위 밖) 카카오톡/이메일 등으로 클라이언트에게 링크 전달
   ↓ "로그아웃" → 로그인 페이지로 이동

4. 견적서 상세 페이지 (클라이언트, 고유 링크로 직접 접근 · 로그인 불필요)
   ↓ 견적서 내용 확인
   ↓ "PDF 다운로드" 클릭 → PDF 파일 다운로드 완료
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명                       | 설명                                                                                                                                                | MVP 필수 이유                                                                  | 관련 페이지        |
| -------- | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ | ------------------ |
| **F001** | 노션 데이터 동기화           | 관리자가 "동기화" 버튼을 클릭하면 Notion API로 지정된 노션 데이터베이스의 견적서 항목(제목, 클라이언트명, 품목, 금액 등)을 가져와 앱 DB에 저장/갱신 | 노션이 견적서의 원본 소스이므로 이 연동이 없으면 서비스 자체가 성립하지 않음   | 관리자 대시보드    |
| **F002** | 견적서 목록 조회             | 동기화된 견적서를 카드/테이블 형태로 목록 표시 (견적서명, 클라이언트명, 금액, 동기화 일시)                                                          | 관리자가 어떤 견적서가 준비됐는지 확인해야 링크를 전달할 수 있음               | 관리자 대시보드    |
| **F003** | 견적서 상세 조회 (고유 링크) | 클라이언트가 고유 링크로 접속하면 로그인 없이 견적서 상세 내용(품목, 수량, 단가, 합계 등)을 웹 페이지로 확인                                        | 클라이언트가 견적서를 확인하는 서비스의 핵심 목적                              | 견적서 상세 페이지 |
| **F004** | PDF 다운로드                 | 견적서 상세 페이지에서 현재 보고 있는 견적서 내용을 PDF 파일로 즉시 다운로드                                                                        | 클라이언트가 견적서를 보관·전달하기 위한 핵심 요구사항                         | 견적서 상세 페이지 |
| **F005** | 고유 링크 발급/복사          | 동기화된 각 견적서마다 추측 불가능한 고유 토큰 기반 링크를 자동 발급하고, 관리자가 목록에서 "링크 복사" 버튼으로 즉시 복사                          | 로그인 없이도 특정 클라이언트만 자신의 견적서에 접근하도록 하는 최소 보안 장치 | 관리자 대시보드    |

### 2. MVP 필수 지원 기능

| ID       | 기능명             | 설명                                                                                                           | MVP 필수 이유                                                                                    | 관련 페이지                    |
| -------- | ------------------ | -------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ | ------------------------------ |
| **F010** | 기본 인증          | 관리자 회원가입/로그인/로그아웃 (관리자는 1인이며 최초 1회만 가입)                                             | 노션 동기화·링크 발급 기능은 사업자 본인만 사용해야 하므로 최소 인증 필요                        | 로그인 페이지, 회원가입 페이지 |
| **F011** | 견적서 데이터 저장 | 노션에서 가져온 견적서 원본 데이터(견적서 정보 + 품목 리스트)를 앱 DB에 캐싱하여 상세 페이지·PDF 생성에 재사용 | 매 조회마다 노션 API를 호출하지 않고 빠르고 안정적으로 견적서를 보여주기 위한 최소 데이터 저장소 | 관리자 대시보드                |

### 3. MVP 이후 기능 (제외)

- 클라이언트 회원가입/로그인 (계정 기반 견적서 이력 관리)
- 견적서 승인/거절, 전자 서명
- 견적서 상태 변경 알림 (이메일/카카오 알림톡 발송)
- 노션 실시간 자동 동기화 (Webhook 기반)
- 견적서 다국어/통화 지원
- 관리자 다중 계정 및 권한 관리
- 견적서 템플릿 커스터마이징, 로고 업로드

## 📄 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `F010` | **메뉴 위치:** 인증 (비로그인 시)

| 항목            | 내용                                                                                                                  |
| --------------- | --------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 관리자 인증 전용 페이지                                                                                               |
| **진입 경로**   | 첫 접속("/") 시 미인증 상태면 자동 리디렉션, 또는 로그아웃 후 이동                                                    |
| **사용자 행동** | 이메일/비밀번호 입력 후 로그인                                                                                        |
| **주요 기능**   | • 이메일/비밀번호 로그인 폼 (React Hook Form + Zod 유효성 검사)<br>• 회원가입 페이지로 이동 링크<br>• **로그인** 버튼 |
| **다음 이동**   | 성공 → 관리자 대시보드, 실패 → 에러 메시지 표시                                                                       |

---

### 회원가입 페이지

> **구현 기능:** `F010` | **메뉴 위치:** 인증 (비로그인 시)

| 항목            | 내용                                                                                                           |
| --------------- | -------------------------------------------------------------------------------------------------------------- |
| **역할**        | 관리자 계정 최초 생성 전용 페이지 (관리자는 1인으로 최초 1회만 사용)                                           |
| **진입 경로**   | 로그인 페이지에서 "회원가입" 링크 클릭                                                                         |
| **사용자 행동** | 이메일/비밀번호 입력 후 계정 생성                                                                              |
| **주요 기능**   | • 이메일/비밀번호 회원가입 폼 (React Hook Form + Zod 유효성 검사)<br>• 중복 이메일 검사<br>• **회원가입** 버튼 |
| **다음 이동**   | 성공 → 로그인 페이지로 자동 이동, 실패 → 에러 메시지 표시                                                      |

### 관리자 대시보드

> **구현 기능:** `F001`, `F002`, `F005`, `F011` | **인증:** 관리자 로그인 필수

| 항목            | 내용                                                                                                                                                                                                                            |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 노션 견적서 동기화 및 클라이언트 전달용 링크 관리의 중심 화면                                                                                                                                                                   |
| **진입 경로**   | 로그인 성공 시 자동 이동, 또는 첫 접속("/") 시 인증된 상태면 자동 리디렉션                                                                                                                                                      |
| **사용자 행동** | 노션 동기화 실행, 동기화된 견적서 목록 확인, 특정 견적서의 링크 복사                                                                                                                                                            |
| **주요 기능**   | • **노션 동기화** 버튼 (Notion API 호출 → 견적서/품목 데이터 갱신)<br>• 견적서 목록 표시 (견적서명, 클라이언트명, 총액, 최근 동기화 일시)<br>• 견적서별 **링크 복사** 버튼 (고유 토큰 URL 클립보드 복사)<br>• **로그아웃** 버튼 |
| **다음 이동**   | 로그아웃 → 로그인 페이지, 링크는 앱 밖(카카오톡/이메일 등)으로 전달                                                                                                                                                             |

### 견적서 상세 페이지

> **구현 기능:** `F003`, `F004` | **인증:** 불필요 (고유 링크 소지자만 접근)

| 항목            | 내용                                                                                                                                                                                             |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **역할**        | 클라이언트가 자신의 견적서 내용을 확인하는 전용 화면                                                                                                                                             |
| **진입 경로**   | 관리자가 전달한 고유 링크로 직접 접속 (로그인/회원가입 불필요)                                                                                                                                   |
| **사용자 행동** | 견적서 내용을 확인하고 PDF로 다운로드                                                                                                                                                            |
| **주요 기능**   | • 견적서 기본 정보 표시 (견적서명, 클라이언트명, 발급일)<br>• 품목별 상세 내역 표시 (품목명, 수량, 단가, 금액, 합계)<br>• 유효하지 않은 링크 접근 시 안내 메시지 표시<br>• **PDF 다운로드** 버튼 |
| **다음 이동**   | PDF 다운로드 완료 (페이지 이동 없음)                                                                                                                                                             |

## 🗄️ 데이터 모델

### Quote (견적서)

| 필드           | 설명                        | 타입/관계       |
| -------------- | --------------------------- | --------------- |
| id             | 고유 식별자                 | UUID            |
| notion_page_id | 원본 노션 페이지 ID         | string          |
| slug           | 클라이언트 접근용 고유 토큰 | string (unique) |
| title          | 견적서명                    | string          |
| client_name    | 클라이언트명                | string          |
| total_amount   | 총 견적 금액                | number          |
| synced_at      | 최근 노션 동기화 일시       | timestamp       |

### QuoteItem (견적 품목)

| 필드       | 설명                    | 타입/관계  |
| ---------- | ----------------------- | ---------- |
| id         | 고유 식별자             | UUID       |
| quote_id   | 소속 견적서             | → Quote.id |
| name       | 품목명                  | string     |
| quantity   | 수량                    | number     |
| unit_price | 단가                    | number     |
| amount     | 품목 금액 (수량 × 단가) | number     |

## 🛠️ 기술 스택 (최신 버전)

### 🎨 프론트엔드 프레임워크

- **Next.js 15.5.3** (App Router + Turbopack) - React 풀스택 프레임워크
- **TypeScript 5** - 타입 안전성 보장
- **React 19.1.0** - UI 라이브러리 (최신 동시성 기능)

### 🎨 스타일링 & UI

- **TailwindCSS v4** (설정파일 없는 새로운 엔진) - 유틸리티 CSS 프레임워크
- **shadcn/ui** (new-york style) - 고품질 React 컴포넌트 라이브러리
- **Radix UI** - 접근성 기반 헤드리스 컴포넌트
- **Lucide React** - 아이콘 라이브러리

### 📝 폼 & 검증

- **React Hook Form 7.x** - 폼 상태 관리
- **Zod** - 스키마 검증 라이브러리
- **Server Actions** - 폼 제출 및 노션 동기화 처리

### 🔗 외부 연동 & 문서 생성

- **@notionhq/client** - Notion API 공식 SDK (견적서 데이터 조회)
- **@react-pdf/renderer** - 견적서 상세 페이지 내용을 PDF로 생성/다운로드

### 🗄️ 백엔드 & 데이터베이스

- **Supabase** - BaaS (관리자 인증, 견적서/품목 데이터 저장)
- **PostgreSQL** - 관계형 데이터베이스 (Supabase 포함)

### 🚀 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼

### 📦 패키지 관리
